# Commit Convention

이 문서는 기존 커밋 기록을 참고해 현재 프로젝트에서 사용하는
커밋 메시지 규칙을 정리한다.

## 기본 형식

- `[TAG] 한 줄 설명`
- 예시:
  - `[FIX] 크롤링 배치 재호출 안정화`
  - `[FEAT] AI 요약 선택 로직 추가`

## 사용 가능한 TAG

- `[FIX]` 버그 수정, 오류 해결
- `[FEAT]` 새로운 기능 추가
- `[REFACTOR]` 리팩터링 (동작 변경 없음)
- `[STYLE]` 포맷/스타일 변경 (기능 변화 없음)
- `[DESIGN]` UI/UX 시각 변경
- `[CHORE]` 빌드/툴링/환경/스크립트 정리
- `[DOCS]` 문서 추가/수정

## 가이드

- 문장은 **간결한 한국어**로 작성한다.
- 한 커밋에는 **하나의 목적**만 담는다.
- 문서 변경만 있으면 `[DOCS]`를 사용한다.
